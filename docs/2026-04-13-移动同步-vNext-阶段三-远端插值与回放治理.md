# 2026-04-13 移动同步 vNext 阶段三：远端插值、回放治理、表现层分离

## 本轮目标

完成此前列出的三个高价值后续项：

1. 远端 snapshot buffer / interpolation window
2. replay-window 治理
3. presentation / animation 更彻底分层

## 本轮完成

### 1. 远端玩家从“单最新快照”升级为真正的 snapshot buffer

修改：

- `clients/bevy_client/src/world/remote_player.rs`

现在 `RemotePlayerState` 不再只保存一个 snapshot，而是：

- 保存一个有界 snapshot ring buffer
- 根据 `server_tick` 建立远端时间线
- 引入 interpolation delay
- 在两帧之间使用插值
- 超出最新 snapshot 时才进行有限外推

此外远端位置采样从：

- `sample_position`

升级为：

- `sample_motion`
  - `position`
  - `velocity`

为表现层和动画层提供更完整输入。

### 2. replay-window 治理正式落地

新增：

- `clients/bevy_client/src/sim/governance.rs`

并修改：

- `clients/bevy_client/src/sim/history.rs`
- `clients/bevy_client/src/sim/reconcile.rs`
- `clients/bevy_client/src/world/local_player.rs`

现在客户端本地 prediction 不再只做“有 ack 就 replay”，而是具备了：

- `ReplayGovernance`
  - `soft_position_error`
  - `hard_snap_distance`
  - `max_replay_frames`
  - `max_pending_inputs`
- `ReplayGovernanceStats`
- `ReplayAction`
  - `Accepted`
  - `Replayed`
  - `HardSnap`
  - `WindowTrimmed`

也就是说：

- correction 太小 -> 不必重放
- correction 很大 -> 允许 hard snap
- replay 窗口过大 -> 触发 window trimming

这让 replay 从“能用”提升到了“可治理”。

### 3. 表现层开始真正拆出 presentation 模块

新增：

- `clients/bevy_client/src/presentation/mod.rs`
- `clients/bevy_client/src/presentation/smoothing.rs`
- `clients/bevy_client/src/presentation/camera.rs`
- `clients/bevy_client/src/presentation/animation.rs`

当前拆分结果：

- `smoothing.rs`
  - 平移平滑
  - scale 平滑
- `camera.rs`
  - 相机目标选择
  - 相机平滑跟随
- `animation.rs`
  - 从 velocity 推导动画输入
  - 基于速度比生成表现 scale

`app.rs` 不再自己承担这些底层 presentation 细节实现。

### 4. 本地/远端表现现在共享统一 presentation 输入思路

`sync_player_visuals` 现在使用：

- 本地玩家：
  - local position
  - local velocity
- 远端玩家：
  - sampled remote motion

统一生成：

- visual translation
- animation-derived scale

这意味着当前即使还是轻量 sprite 客户端，也已经不再是“网络状态直接推动画面”的耦合写法。
