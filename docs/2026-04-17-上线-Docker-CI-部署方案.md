# 2026-04-17 上线 — Docker + CI → 阿里云 ACR 部署方案

## 0. 目标

- 客户端能在真实网络条件下连通服务端（延迟、丢包、NAT 暴露出来）
- 一次 `git push`，CI 自动打镜像 → 推送阿里云 ACR
- 服务器侧 `docker compose pull && docker compose up -d` 完成升级
- 单容器 MVP（本期不做分布式集群），PostgreSQL 与 app 同栈 compose
- 域名上 HTTPS，gate_server 裸 TCP

## 2. 容器拓扑（生产）

### 2.1 服务清单

| 容器 | 镜像 | 暴露端口 | 备注 |
|------|------|---------|------|
| `app` | `registry.cn-xxx.aliyuncs.com/NS/ex_mmo_cluster:TAG` | 宿主 127.0.0.1:${AUTH_PORT}/${VISUALIZE_PORT} + 宿主 ${GATE_TCP_PORT} | umbrella 打包后的 Elixir release |
| `postgres` | `postgres:16-alpine` | 内网 5432 | 数据卷 `pg_data` |
| **宿主 nginx** | （非容器） | 80/443 | 反代到 127.0.0.1:${AUTH_PORT}/${VISUALIZE_PORT}，证书由 certbot 管 |

### 2.2 端口映射

| 流向 | 宿主端口 | 目标 | 协议 | TLS 终端 |
|------|---------|------|------|----------|
| `auth.DOMAIN` | 443 | 127.0.0.1:${AUTH_PORT} → app:4000 | HTTPS | 宿主 nginx |
| `game.DOMAIN` | 443 | 127.0.0.1:${VISUALIZE_PORT} → app:4001 | HTTPS + WS | 宿主 nginx |
| `gate.DOMAIN` | `${GATE_TCP_PORT}` | app:29000 | TCP 明文（自定义二进制） | 暂不终端（MVP） |
| 管理/pg | — | — | — | **不暴露宿主** |

**关于 gate 的 TLS**：MVP 阶段走明文 TCP，避免引入 nginx stream + TLS 或 stunnel 的复杂度。
客户端协议本身是自定义二进制，账户密码已经由 auth_server 走 HTTPS 发放 token，
gate 只接 token 验证后的会话，风险可控。**若公网压测发现需要加密，再上 TLS 1.3 前置层**。

### 2.3 网络

- 内部网络 `ex_mmo_cluster_internal`（bridge），`postgres` 不暴露宿主
- `app` 通过服务名 `postgres` 连接数据库
- HTTP 端口绑 127.0.0.1，**只对宿主 nginx 可见**；nginx 终结 TLS 后反代到 loopback

## 3. 镜像设计（Dockerfile）

采用三阶段构建：

```
Stage 1: rust builder          (Rust 1.94 + build-essential)
  ↓ 编译 native/*/target/release/*.so
Stage 2: elixir builder        (Elixir 1.18.4 + OTP 28 + Node for asset build)
  ↓ mix deps.get → deps.compile → tailwind/esbuild → mix release
Stage 3: runtime               (debian:bookworm-slim + libstdc++)
  ↓ 仅复制 _build/prod/rel/ex_mmo_cluster/
```

**Stage 1 - Rust builder**
- base: `rust:1.94-bookworm`
- 产物：四个 `.so` 文件（`scene_ops`, `octree`, `coordinate_system`, `movement_engine`）
- 这一 stage 独立是为了让 Rust 编译缓存独立（~5 min 首次，后续命中 cache）

**Stage 2 - Elixir builder**
- base: `hexpm/elixir:1.18.4-erlang-28.3.1-debian-bookworm-20250908-slim`
- 环境：`MIX_ENV=prod`
- 步骤：
  1. `mix deps.get --only prod`
  2. `mix deps.compile`
  3. 复制 Stage 1 的 `.so` 到 `apps/scene_server/priv/native/`
  4. `mix assets.deploy`（Phoenix 1.8 asset pipeline，合并 `tailwind` + `esbuild` + `phx.digest`）
  5. `mix release ex_mmo_cluster`

**Stage 3 - Runtime**
- base: `debian:bookworm-slim`
- 安装：`libstdc++6 ncurses-bin openssl ca-certificates libssl3 locales`
- 用户：`ex_mmo_cluster:ex_mmo_cluster`（非 root）
- 工作目录：`/app`
- 复制：`COPY --from=builder /app/_build/prod/rel/ex_mmo_cluster /app`
- entrypoint: `/app/bin/ex_mmo_cluster start`

### 3.1 Release 产物结构

```
/app/
├── bin/ex_mmo_cluster              # 入口脚本
├── lib/
│   ├── gate_server-0.1.0/
│   ├── auth_server-0.1.0/
│   ├── scene_server-0.1.0/
│   │   └── priv/native/*.so     # Rust NIF
│   └── ...
├── releases/0.1.0/
│   ├── env.sh.eex
│   ├── runtime.exs
│   └── vm.args.eex
└── erts-15.x/                    # ERTS 打包进镜像（include_erts: true）
```

**选型决策**：

- ✅ `include_erts: true` — 镜像自带 ERTS，runtime 不需要装 Elixir/Erlang
- ✅ 单 release `:ex_mmo_cluster` 包含所有 umbrella app（除 legacy `data_store` / `data_contact`）
- ❌ 不做 hot code reload（MVP 靠重启升级）
- ❌ 不做多架构（仅 `linux/amd64`）

## 4. 配置与环境变量

### 4.1 变量清单

| 变量 | 必填 | 默认 | 说明 |
|------|------|------|------|
| `SECRET_KEY_BASE` | ✅ | — | Phoenix session 签名，`mix phx.gen.secret` 生成 |
| `RELEASE_COOKIE` | ✅ | — | Erlang 节点 cookie，单容器也要固定，`openssl rand -base64 48` |
| `PHX_SERVER` | ✅ | `true` | 启动 HTTP 监听 |
| `PHX_HOST` | ✅ | — | 公网域名，如 `auth.example.com`（Phoenix url: 用） |
| `MMO_DB_HOST` | ✅ | `postgres` | compose 服务名 |
| `MMO_DB_PORT` | — | `5432` | |
| `MMO_DB_NAME` | ✅ | `ex_mmo_cluster_prod` | |
| `MMO_DB_USER` | ✅ | — | |
| `MMO_DB_PASSWORD` | ✅ | — | `openssl rand -base64 24` |
| `MMO_DB_POOL_SIZE` | — | `10` | |
| `AUTH_PORT` | — | `4000` | auth_server HTTP |
| `VISUALIZE_PORT` | — | `4001` | visualize_server HTTP |
| `GATE_PORT` | — | `5555` | gate_server TCP（TODO：从 runtime 读取）|
| `DISABLE_CLUSTER` | — | `true` | MVP 单容器禁用 libcluster gossip |
| `RELEASE_DISTRIBUTION` | — | `none` | BEAM 不起分布式 |

### 4.2 Config 适配

- `config/config.exs` 的 `libcluster` gossip 在 prod 要条件化：`if !System.get_env("DISABLE_CLUSTER"), do: ...`
- 或在 `config/prod.exs` 显式覆盖为 `topologies: []`
- `config/runtime.exs` 的 prod 分支已有 `PHX_HOST`、`SECRET_KEY_BASE` 处理，需补 gate_server 端口和 cookie 注入

### 4.3 节点启动模式

**MVP（单容器，不分布式）**：
```
RELEASE_DISTRIBUTION=none
RELEASE_NODE=ex_mmo_cluster@127.0.0.1
```

`beacon_server` / `libcluster` / `Horde` 在单节点场景退化为空操作；本地 `:pg` 仍然工作。

**后续（多容器分布式）**：改为 `sname` 模式 + overlay network + `DNS_CLUSTER_QUERY`。

## 5. docker-compose.yml 要点

```yaml
services:
  postgres:
    image: postgres:16-alpine
    restart: unless-stopped
    environment: [POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB]
    volumes: [pg_data:/var/lib/postgresql/data]
    networks: [internal]
    healthcheck: pg_isready
    # 不暴露宿主端口

  app:
    image: ${IMAGE_TAG}
    restart: unless-stopped
    env_file: .env
    depends_on: { postgres: { condition: service_healthy } }
    ports:
      # HTTP 端口绑 loopback，仅宿主 nginx 可达
      - "127.0.0.1:${AUTH_PORT}:4000"
      - "127.0.0.1:${VISUALIZE_PORT}:4001"
      # gate TCP 直接对外（自定义二进制协议，nginx http 模块不能反代）
      - "${GATE_TCP_PORT}:29000"
    networks: [internal]

volumes: { pg_data: {} }
networks: { internal: {} }
```

### 5.1 首次部署 vs 升级

- **首次**：`docker compose up -d` + 首次启动跑 `docker compose exec app /app/bin/ex_mmo_cluster eval 'DataService.Release.migrate()'`
- **升级**：`docker compose pull && docker compose up -d app`（仅 app 重启，pg 不动）

`DataService.Release.migrate/0` 需新增，调用 `Ecto.Migrator` 在 release 内跑 migration（不依赖 Mix）。

## 6. nginx 反向代理（宿主侧）

完整模板见 `deploy/nginx.conf.example`。核心 server block：

```nginx
server {
    listen 443 ssl http2;
    server_name auth.${DOMAIN};
    ssl_certificate     /etc/letsencrypt/live/auth.${DOMAIN}/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/auth.${DOMAIN}/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:4000;
        proxy_http_version 1.1;
        proxy_set_header Host              $host;
        proxy_set_header X-Real-IP         $remote_addr;
        proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        # Phoenix LiveView / Channels
        proxy_set_header Upgrade           $http_upgrade;
        proxy_set_header Connection        "upgrade";
        proxy_read_timeout 3600s;
    }
}
# game.${DOMAIN} 同理 → 127.0.0.1:4001
```

证书由宿主侧 `certbot --nginx -d auth.${DOMAIN} -d game.${DOMAIN}` 自动签发并续期；
不再依赖容器内的 ACME 客户端。

**gate TCP** 不走 nginx —— compose 直接将宿主 `${GATE_TCP_PORT}` 映射到 `app:29000`，
客户端解析 `gate.{$DOMAIN}` 到宿主 IP，连接 `gate.{$DOMAIN}:${GATE_TCP_PORT}`。

## 7. CI 工作流

新增 `.github/workflows/docker-publish.yml`，与现有 `ci.yml`（单 app 测试）**并存**：

- `ci.yml`：PR + push 都跑，只做测试
- `docker-publish.yml`：仅 `master` push + 语义版本 tag 触发，打镜像推 ACR

### 7.1 关键 steps

1. `actions/checkout@v4` with `submodules: true`
2. `docker/setup-buildx-action@v3`
3. `docker/login-action@v3` → `registry.cn-<region>.aliyuncs.com`
4. `docker/metadata-action@v5` → 生成 tags（sha, branch, latest, semver）
5. `docker/build-push-action@v6`
   - `context: .`
   - `platforms: linux/amd64`
   - `cache-from: type=gha`
   - `cache-to: type=gha,mode=max`
   - `push: true`

### 7.2 Secrets 清单（仓库 settings → secrets → actions）

| Secret | 示例值 |
|--------|--------|
| `ALIYUN_REGISTRY` | `registry.cn-hangzhou.aliyuncs.com` |
| `ALIYUN_NAME_SPACE` | `your-namespace` |
| `ALIYUN_REGISTRY_USER` | `aliyun@1234567890` |
| `ALIYUN_REGISTRY_PASSWORD` | （阿里云 ACR 独立密码） |

## 8. 服务器侧准备

见 `docs/2026-04-17-服务器部署手册.md`（另文档）。要点：

1. Ubuntu 22.04/24.04 装 `docker-ce` + `docker-compose-plugin`
2. 配置 ACR 登录凭证（`docker login registry.cn-xxx.aliyuncs.com`，复用 CI 密码或创建只读账号）
3. 放 `docker-compose.yml` + `.env` + `nginx.conf.example`（部署到 `/etc/nginx/sites-available/`）到 `/opt/ex_mmo_cluster/`
4. 初始化 pg + 跑 migration
5. `docker compose up -d`
6. 客户端连接 `game.{$DOMAIN}:${GATE_PORT}` 验证

## 9. 风险与开放项

| # | 项 | 当前取舍 | 风险 |
|---|----|---------|------|
| R1 | libcluster 在单容器里不起 | 配置关闭，但代码仍在 | 后续多容器要回来改 |
| R2 | ERTS + rapier3d-f64 + BEAM 总镜像体积 | 预估 250-350 MB | 可接受，不优化 |
| R3 | CI Rust 编译 ~5 min | `cache-to gha` + cargo registry cache | 首次推送较慢，后续 < 2 min |
| R4 | gate TCP 明文 | MVP 接受 | 上线后观察是否需要 TLS |
| R5 | 单容器单点 | MVP 接受，停机升级短暂 | 业务允许的前提 |
| R6 | PostgreSQL 同栈 | 方便起步 | 生产最终应切托管（阿里云 RDS） |
| R7 | `visualize_server` 监控面板暴露在公网 | 需要加 basic auth 或 IP 白名单 | 初次上线前必须处理 |
