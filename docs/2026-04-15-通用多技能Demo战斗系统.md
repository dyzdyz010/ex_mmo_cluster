# 2026-04-15 通用多技能 Demo 战斗系统

## 本轮目标

把当前 demo 从“单技能闭环”升级成：

- 数据驱动技能 catalog
- 通用 effect 执行器
- 多种技能类型
- 可复制的视觉 cue
- stdio / observe / E2E 可验证

## 新增能力

### 1. 通用技能结构

新增：
- `Combat.CastRequest`
- `Combat.EffectSpec`
- `Combat.EffectEvent`
- `Combat.Executor`

`Combat.Skill` 现在不再只是一个简单的 `{damage, radius, cooldown}` 结构，而是描述：
- target_mode
- range
- delivery_kind
- projectile_speed
- cast cue
- effect spec 列表

### 2. 支持的技能类型

- `1 Arc Slash`：近战瞬时单体
- `2 Arc Bolt`：远程单体 projectile
- `3 Target Ring`：目标中心 AOE
- `4 Cascade Sigil`：projectile 命中后触发 AOE + chain arcs
- `101 Slime Spit`：训练 NPC 的 ranged projectile

### 3. 复制策略

新增 `EffectEvent` 作为 authoritative gameplay cue：
- melee arc
- projectile
- aoe ring
- chain arc
- impact pulse

状态仍由：
- `CombatHit`
- `PlayerState`

承载。

也就是：
- 状态正确性与视觉 cue 分开传播
- 客户端不根据 HP 变化猜测视觉，而是播放服务器给的 cue

### 4. 客户端支持

新增：
- `Tab` 循环选择目标
- `RMB` 选择地面目标点
- `skill <id> [target_cid]` stdio 定向施法
- `target_point <x> <y> [z]` / `clear_target_point` stdio 点选目标
- NPC 与玩家共享远端移动，但通过 effect cue 播放不同技能表现

客户端视觉现在支持：
- melee arc
- projectile travel
- aoe ring
- chain arc
- impact pulse

## 最佳实践说明

本轮采用的做法不是临时发明，而是接近行业常见思路：

- 服务器权威决定 cast / hit / damage / follow-up
- projectile travel time 由服务器决定结算时间
- 客户端只消费 stateless gameplay cue 来做视觉
- 远端自然度依赖：
  - 已有 remote snapshot interpolation
  - 新增 effect cue playback

这类似于：
- Valve 关于 authoritative hit / projectile 区分的思路
- Unreal GAS 中 gameplay result 与 GameplayCue 分离的思路
- Unity netcode 中 authority state 与 presentation timeline 分离的思路

## 验证

通过：
- `mix test`
- `cargo test`
- `mix demo.run --smoke --exit-after 20 --bot-count 2 --output_dir .demo/smoke-multiskill-3 --observe_dir .demo/smoke-multiskill-3`
- `powershell -ExecutionPolicy Bypass -File .\scripts\e2e-stdio.ps1 -ObserveDir .demo/e2e-multiskill-3 -BotCount 1`

## E2E 证明到的点

### client stdio
- 依次成功发出 skill 1/2/3/4
- 能看到 NPC 列表
- 能看到目标玩家列表

### server stdio
- 能看到 NPC 初始状态
- 能看到 NPC 被击杀后的状态
- 能看到 NPC respawn 后再次参与战斗
- 能看到目标玩家状态变化

### client observe
- 记录到了：
  - `effect_event` 的 `MeleeArc`
  - `Projectile`
  - `AoeRing`
  - `ChainArc`
- 记录到了不同 skill id 对应的 `CombatHit`
- 记录到了 NPC skill `101` 与玩家 skill `1/2/3/4` 同时存在

## 当前结论

这一轮之后，demo 已经不是“单技能演示”，而是：

> **一个具备通用技能结构、通用效果执行器、通用视觉 cue 复制的多技能 authoritative demo。**

这已经可以继续承载更复杂的 business/content 抽象，而不需要回到“一技能一模块”的方向。
