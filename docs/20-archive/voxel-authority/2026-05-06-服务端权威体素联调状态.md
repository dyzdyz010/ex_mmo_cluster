# 2026-05-06 服务端权威体素联调状态

本文记录本轮浏览器客户端与 Elixir 服务端权威体素链路的当前实现状态、已修复问题和可复现验收入口。

## 当前运行约定

- 本地服务端固定使用 20000 号段：
  - `AUTH_PORT=20000`
  - `VISUALIZE_PORT=20001`
  - `GATE_TCP_PORT=20002`
  - `GATE_UDP_PORT=20003`
- 浏览器客户端默认运行在 `http://127.0.0.1:5173/`。
- 浏览器 HTTP auth / dev_seed 默认走 Vite `/ingame` 同源代理，WebSocket 使用 `/ingame/ws`。
- 默认网页模式为：
  - `VITE_MOVEMENT_TRANSPORT=server`
  - `VITE_VOXEL_SYNC=online`
  - `VITE_VOXEL_DEV_SEED=1`
  - `VITE_VOXEL_PRIME_DEMO_BLOCK=0`

## 已修复问题

### 登录和 dev_seed 过慢

旧实现会在每次 `dev_seed` 时逐格写入 256 个宏格，并触发多次 Scene/DataService 持久化。刷新页面时即使地形已经存在，也会重复走慢路径。

当前实现：

- `WorldServer.Voxel.DevSeed` 仍通过 `WorldServer.Voxel.MapLedger` 续租和发布写 token。
- starter terrain 通过 `SceneServer.Voxel.ChunkDirectory.apply_intents/2` 批量写入同一 chunk。
- `SceneServer.Voxel.ChunkProcess.apply_intents/2` 对同一 chunk 做一次批量提交，只有实际变化时才持久化。
- 已存在的相同实体块会被判定为 no-op，计入 `skipped`。

实测：

- 首次新轴向 seed：`attempted=256 written=256 skipped=0 max_chunk_version=1`。
- 后续刷新 / doctor：`attempted=256 written=0 skipped=256`。
- `./scripts/dev-doctor.sh` 复跑总耗时约 1.4 秒。

### prefab 鼠标右键无效

旧实现中，在线热栏仍暴露离线 refined prefab（如 sphere/cylinder/stairs），而服务端 v1 catalog 只认识 `builtin_pillar_3`、`builtin_floor_3x3`、`builtin_cube_2x2x2`。同时在线 boundary snap 返回 `server_authority_not_supported` 后没有正确 fallback 到服务端 `PrefabPlaceIntent`。

当前实现：

- 在线热栏第 5/6/7 格为服务端 v1 prefab：pillar / floor / cube。
- 右键命中体素表面后，在线 boundary snap 的 `server_authority_not_supported` 会记录 `prefab_boundary_snap_fallback`，然后提交 `PrefabPlaceIntent`。
- 成功收到 `VoxelIntentResult accepted` 后清理 `lastError`，避免 HUD/Panel 显示历史假失败。

实测：

- 鼠标右键 pillar：`prefab_place origin="6,1,6"`。
- 服务端返回：`intent_result_applied result_code="accepted"`。
- 下行增量：同一 chunk 收到三条 `chunk_delta_received`，版本推进到 7。
- CLI 读回：`cell 6 1 6`、`cell 6 2 6`、`cell 6 3 6` 均为实体块。

### 登录后出生点没有地面

旧实现的 starter terrain 使用了错误轴向，视觉上不是浏览器 Y 轴竖直的地面。另一个隐蔽问题是在线模式继承了离线 `bootstrap()`，会在客户端本地种 showcase 地形，导致观察结果混入非服务端数据。

当前实现：

- `DevSeed` 在 chunk `(0,0,0)` 的 `y=0` 宏格层写入 `16 x 16` stone platform。
- 默认角色出生点为服务端移动坐标 `{x=750,y=750,z=100}`，浏览器映射为 `{x=750,y=100,z=750}`，位于该平台上方。
- `OnlineVoxelWorldAdapter.bootstrap()` 不再写入离线 showcase；在线 `WorldStore` 初始为空，只接受服务端 snapshot/delta/result。

实测：

- 刷新后仅有服务端订阅的 `chunks=27`。
- 初始服务端平台和已有 prefab 后 `solidBlocks=259`，右键 pillar 后 `solidBlocks=262`。
- 本地玩家状态：`position=750.0,100.0,750.0`，`movementMode=grounded`，渲染显示中心 `y=160`。

## 当前注意事项

- Playwright 无头/自动化环境可能拒绝 pointer lock；这只影响相机拖动体验，不影响 CLI 和真实浏览器操作入口。验证鼠标放置时可先拖动相机让中心准星命中地面，再右键。
- 在线模式下 refined micro prefab 的本地功能仍保留在 `VITE_VOXEL_SYNC=offline`，服务端 v1 只支持当前三种 macro blueprint。
- `mix precommit` 只在 Phoenix 子应用 `auth_server` / `visualize_server` 内定义；umbrella 根没有同名 alias。本轮使用 targeted mix/npm 验证和 dev-doctor 覆盖。
