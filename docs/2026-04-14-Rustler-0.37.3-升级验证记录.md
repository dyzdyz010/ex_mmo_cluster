## 过程中的真实发现

这次升级验证并不只是“版本号改掉 + 单测通过”。在更重的 smoke 负载下，发现了一个真实运行时问题：

### 现象

在：

- 2 个 demo bot
- UDP fast-lane
- movement/chat/skill 并发

的 smoke 路径中，gate 内部对 scene/player 的同步 `GenServer.call` 默认超时容忍不足，导致：

- generic result error
- 一度触发 UDP acceptor 重启风险

### 处理

修正了：

- `apps/gate_server/lib/gate_server/worker/tcp_connection.ex`
- `apps/gate_server/lib/gate_server/worker/udp_acceptor.ex`

改为：

- 对 scene/player 内部调用使用更合理的超时
- UDP acceptor 使用安全调用路径，避免因 call timeout 直接把共享 UDP worker 打崩

### 结论

因此，这次升级最终不只是“依赖升级完成”，而是：

> **依赖升级 + post-upgrade runtime hardening 一并完成。**
