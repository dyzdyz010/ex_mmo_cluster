# Chat v1 Implementation Plan

**Goal:** Move player chat out of Scene AOI and establish a standalone,
CLI-observable MMO chat runtime.

**Architecture:** Gate authenticates and forwards. Chat owns channel membership,
delivery planning, message IDs, bounded history, and observe logs. Scene no
longer owns the Gate chat send path. World partition remains the future source
for `region_id` and `chunk_coord`.

## Completed Slice

- [x] Add `apps/chat_server` umbrella app.
- [x] Add `ChatServer.Runtime` session table, validation, message IDs, bounded
      history, and fan-out to Gate connection processes.
- [x] Add `ChatServer.DeliveryPlan` pure planner for `world`, `region`, `local`,

- [x] Add `ChatSayScoped(0x0A)` for client-requested `world` / `region` /

      `window.__voxelCli.run("chat <world|region|local> <text...>")`, `0x0A`
      encoder, `0x89` decoder, and chat observe/debug snapshots. The browser still
      sends only scope/text; Gate derives the concrete channel.
- [x] Add server-derived candidate-region preselection for local chat when Gate

      narrow candidates, then still applies exact server-side chunk-radius
      filtering.
- [x] Add `ChatServer.RuntimeDirectory` and `mix chat_server.shard_observe` as
      the logical-scene shard entry point. The directory owns routing metadata
      only; shard-local `ChatServer.Runtime` processes still own sessions,
