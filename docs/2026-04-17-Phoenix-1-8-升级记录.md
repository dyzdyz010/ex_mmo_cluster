# Phoenix 1.8 升级记录

- **日期**：2026-04-17
- **分支**：`phoenix-1-8-upgrade`
- **影响 App**：`auth_server`、`visualize_server`
- **升级目标**：从 Phoenix 1.6 + LiveView 0.17 升级到 Phoenix 1.8 + LiveView 1.1，使用 `mix phx.new` 生成的模板作为基线。

## 策略

采用 **模板重生 + 业务迁移** 的方式，而不是逐条手动 diff 旧项目：

1. 归档旧的 Phoenix 1.6 版本到 `archive/` 目录（commit `35a58b0`）。
2. 使用最新 `mix phx.new` 为每个应用重新生成骨架（保持单 app 形式，不拆 `_web`）。
3. 将旧项目中的业务模块、路由、模板、worker 逐项移植到新骨架上，按 Phoenix 1.8 的写法改写接口。
4. 调整 umbrella `config/*.exs` 以容纳两个新 Endpoint 的配置。
5. 跑完测试，确认无回归后归档锁定。

> 不做的事：保留任何 1.6 时代的 `*.eex`、`Routes.xxx_path`、`use Phoenix.Controller` + 独立 view 模块的写法。

## 关键技术栈升级

| 组件 | 之前 | 之后 |
|------|------|------|
| Phoenix | 1.6 | 1.8.5 |
| Phoenix LiveView | 0.17 | 1.1.28 |
| HTTP 适配器 | Cowboy | Bandit 1.5 |
| 模板 | `*.eex` + 独立 view 模块 | `*.heex` + `embed_templates`，`<Layouts.app>` |
| 路由辅助 | `Routes.xxx_path(@conn, ...)` | `~p"/..."` verified routes |
| 表单 | `form_for` + changeset | `<.form>` + `to_form` |
| 集群 | 仅 libcluster | libcluster + DNSCluster（1.8 模板默认） |
| esbuild / tailwind | 旧版 | esbuild 0.25.4 / tailwind 4.1.12 |
| 图标 | 自定义 | heroicons（1.8 模板自带 core_components） |

## 业务迁移清单

### auth_server

保留且 **未改变对外 API** 的模块（关键跨 app 契约）：

- `AuthServer.AuthWorker`（`gate_server` 测试直接依赖）
  - `build_session_claims/2`、`issue_token/1`、`verify_token/1`
  - `validate_username/2`、`validate_cid/2`
  - `authorize_character/2`、`fetch_authorized_character/2`
  - `@token_salt`、`@max_age` 等常量保持
  - `Phoenix.Token.sign/verify` 继续传 `AuthServerWeb.Endpoint`，仅换了 Endpoint 的实现
- `AuthServer.Accounts`：`find_by_username/1`、`character_owned_by_account?/2`，依然走 `dispatch_to_data_service/1`
- `AuthServer.Interface`（GenServer）：通过 `BeaconServer.Client.join_cluster/register/await` 注册并解析 `data_contact` 节点
- `AuthServer.InterfaceSup`：单子包装 `AuthServer.Interface`

改造：

- Controller / HTML：`IngameController` + `IngameHTML`，使用 `embed_templates "ingame_html/*"`；路径由 `Routes.ingame_path(...)` 改为 `~p"/ingame/login_success?#{[code: code]}"`；模板改用 `<Layouts.app flash={@flash}>` + `<.form action={~p"/ingame/login_post"}>`。
- `Application.start/2`：在 1.8 骨架基础上追加 `interface_children()`，根据 `Application.get_env(:auth_server, :start_interface?, Mix.env() != :test)` 决定是否启动 `AuthServer.InterfaceSup`，让测试环境不启动集群接口。
- `mix.exs`：在 phx.new 默认依赖上追加 `:dns_cluster`、`:bandit`、`:bcrypt_elixir`、`:libcluster`，以及 umbrella 内部 `{:data_init, in_umbrella: true}`、`{:data_service, in_umbrella: true}`、`{:beacon_server, in_umbrella: true}`。

### visualize_server

- `VisualizeServerWeb.SceneLive.Index`：1.8 风格 LiveView。`mount/3` 读 `:scene_node` 配置并在 `connected?(socket)` 时启动 `@tick_interval = 1_000` 心跳；`handle_info(:data_update, ...)` 通过 `push_event(socket, "data", ...)` 下发数据。
- 对外通信仍采用分布式 Erlang：`GenServer.call({SceneServer.PlayerManager, scene_node}, :get_all_players, 500)` 和 `{:ok, {x, y, _z}} = GenServer.call(pid, :get_location, 500)`。保留 `safe_call/2`（`catch :exit, _ -> :error`）避免远端节点波动拖垮 LiveView。
- 模板 `index.html.heex`：包裹 `<Layouts.app flash={@flash}>`，`#scene-visualizer` 仍带 `phx-update="ignore"`，保留 JS 控制权。
- Router：`scope "/"` 下追加 `live "/scene", SceneLive.Index, :index`。

## 配置收敛（`config/*.exs`）

`config.exs` 中集中声明两个 Endpoint、两套 esbuild/tailwind sub-key、`DNSCluster` 占位（`dns_cluster_query`）、libcluster gossip、`data_service` Ecto 配置。

关键点：

- `config :phoenix_live_view, :colocated_js, disable_symlink_warning: true`：Windows 上 LiveView ColocatedJS 创建符号链接会抛 `:eperm`，此开关只禁用告警本身。
- `config :libcluster, topologies: []`（在 `test.exs`）：umbrella 测试环境统一禁用 gossip topology，避免 Windows 上 UDP 端口 `:eaddrinuse`。
- `runtime.exs` 拆分出 `AUTH_PORT`（默认 4000）、`VISUALIZE_PORT`（默认 4001），两个 Endpoint 分别监听。`PHX_SERVER=true` 时两者同时进入 server 模式。
- 测试端口：`auth_server` 4002，`visualize_server` 4003。

## 验证结果

- `cd apps/auth_server && mix compile --warnings-as-errors`：clean。
- `cd apps/visualize_server && mix compile --warnings-as-errors`：clean。
- `cd apps/auth_server && mix test`：5 tests, 0 failures。
- `cd apps/visualize_server && mix test`：5 tests, 0 failures。
- `cd apps/beacon_server && mix test --no-start`：1 doctest + 10 tests, 0 failures。
- `cd apps/gate_server && mix test --no-start`：101 tests pass。28 个 invalid 来自 `TcpConnectionProtocolTest` 的 `setup_all`（需要 PostgreSQL 在 5432 监听，本地环境未起库），与 Phoenix 升级无关。
- 使用 `AuthServer.AuthWorker` 的 gate_server 测试在编译期已解析 API，跨 app 契约保留。

> `mix compile --warnings-as-errors` 在 umbrella 根目录仍会失败，但唯一警告来自 `data_init/lib/table_def.ex` 里 `memento 0.3.2` 的 `Range.new/2` 遗留告警，属于老代码噪音，不是 1.8 迁移引入。

## 给后续升级的提示

- Phoenix 1.8 Endpoint 使用 `persistent_term` 保存实例，`mix test --no-start` 会让 `~p""` verified routes 失败；对于带 Endpoint 的 app 必须以正常模式跑测试（或者在 `test_helper.exs` 里手动启动 Endpoint）。
- `phx.new` 会覆盖仓库根部的 `AGENTS.md`、`config/config.exs` 等；再生成后务必重新合并，不要直接丢弃项目自身的内容。
- Windows 下继续使用 VS Dev Command Prompt + `HEX_HTTP_CONCURRENCY=1` + `HEX_HTTP_TIMEOUT=120`，并关闭 LiveView colocated symlink 告警。
