## 路线 C 三项增量

### C.1 Jitter-adaptive soft threshold（抖动自适应软阈值）

**问题**：`soft_position_error = 2.0` 是常量。RTT 稳定时合理，RTT 抖动时会频繁触发 Replay。
**参考**：Overwatch GDC 2017 披露过类似做法；Source Engine 早期也有类似思路。
**算法**：

```
jitter_ms = EWMA(|RTT_sample - prev_RTT|, α=0.15)
effective_soft = clamp(
    base_soft + k_jitter * jitter_ms + k_rate * recent_replay_rate,
    base_soft,
    max_soft
)
```

**参数建议**：`base_soft=2.0 u`，`max_soft=8.0 u`，`k_jitter=0.02 u/ms`，`k_rate=0.0`（先不开，观察后再加）。

### C.2 correction_flags 语义化分派（优先级最高）

**问题**：服务端物理把玩家推开、玩家被传送、反作弊踢回——这些在客户端看起来都是"预测失败"，当前统一走距离阈值判定，会把被推挤的场景错误地当预测误差 Replay，或把小位移的 TELEPORT 误判为 Accepted。

**位定义**（在 `movement_core::ack` 固化，两端共用）：

| 位 | 名 | 触发场景 | 客户端处理 |
|----|-----|-----|----|
| `0x01` | `TELEPORT` | 脚本传送、复活、跨场景 | HardSnap 清空所有历史 |
| `0x02` | `COLLISION_PUSH` | 物理推挤（墙/怪/击退） | 强制 Replay 分支，忽略 soft 阈值（距离再小也不当 Accepted） |
| `0x04` | `STATUS_OVERRIDE` | 状态覆盖（晕眩速度归零、buff 加速） | 用 ack 的 velocity/mode 覆盖当前，不回放未来输入 |
| `0x08` | `ANTI_CHEAT_REJECT` | 反作弊判定越界 | HardSnap + 打日志 + 上报统计 |
| `0x10..` | 保留 | | |

**发射端**（`scene_server/movement/engine.ex`）：
- 在 NIF step 后对比 `frame.input_dir` 与 `next_state.velocity` 的夹角；超过 90° 或 |Δv| 显著 → OR 上 `COLLISION_PUSH`
- 外部传入（PlayerCharacter 的 teleport 逻辑）→ OR 上 `TELEPORT`

**消费端**（`clients/bevy_client/src/sim/reconcile.rs`）：
- 在距离分支之前新增 flags 分派，各 flag 对应独立动作，互不重叠
- 产出 `ReplayAction::ForcedReplay` / `ReplayAction::Teleport` 区分于距离触发的 HardSnap / Replayed

### C.3 分层同步策略（大部分已在线上）

**现状盘点**：
- 本地玩家：CSP+SR ✅
- 远程玩家：Dead Reckoning + Hermite 插值 + 250ms 外推 ✅
- 缺口：远程 snapshot 无法表达 TELEPORT 语义。目前 `RemoteMoveSnapshot` 只有 position/velocity/acceleration/mode，远程玩家被传送时插值会造成"滑动过去"的视觉错误

**本轮**：
- `RemotePlayerState::push_snapshot` 接受 optional `CorrectionFlags`，TELEPORT 位 → 清空 buffer 重新 seed
- 线协议暂不扩展（PlayerMove 帧 70B 不动），留接口给后续协议升级

## 实施顺序与验收

| Sprint | 范围 | 预估复杂度 | 验收 |
|--------|------|-----------|------|
| **S1 = C.2** | 端到端 flags 语义 | 中 | 新单测 ≥ 3 条；现有测试全绿；服务端/客户端典型 collision/teleport 场景视频稳定 |
| **S2 = C.1** | 抖动自适应 | 低-中 | jitter 估计器单测；抖动网络下 Replay 频率下降可量化 |
| **S3 = C.3** | 远程 TELEPORT 接口 | 低 | `RemotePlayerState` 新测；不改线协议 |

**回归基准**：
- `cargo test` in `clients/bevy_client/`
- `cd apps/scene_server && mix test --no-start`
- `cd apps/gate_server && mix test --no-start`

## 不做的事

- 不推翻 CSP+SR 换 rollback netcode（B 路线）
- 不改 PlayerMove / MovementAck 线格式字节数
- 不把 remote interpolation 改成 full physics replay（代价不值）
- 不加服务端权威的 latency compensation / lag compensation（留给 PvP 阶段）
