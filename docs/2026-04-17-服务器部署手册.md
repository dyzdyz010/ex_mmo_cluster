# 2026-04-17 服务器部署手册（Hemifuture MMO — 单容器 MVP）

对应架构文档：`docs/2026-04-17-上线-Docker-CI-部署方案.md`

## 0. 前提

- 一台 Ubuntu 22.04 或 24.04 的阿里云/国内服务器（≥ 4C 8G）
- 已解析到该服务器的域名 `$DOMAIN`，至少三条 A 记录：
  - `auth.$DOMAIN`
  - `game.$DOMAIN`
  - `gate.$DOMAIN`（TCP 用，也可以直接用 IP）
- 服务器安全组放通：`80/tcp`、`443/tcp`、`${GATE_TCP_PORT}/tcp`（默认 29000）
- 阿里云 ACR 账号已可登录（CI 推送用的相同凭证即可）

## 1. 首次初始化

### 1.1 装 Docker 与 compose 插件

```bash
# Ubuntu 官方源
sudo apt-get update
sudo apt-get install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
    -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] \
    https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo $VERSION_CODENAME) stable" | \
    sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io \
    docker-buildx-plugin docker-compose-plugin

# 当前用户加入 docker 组（避免每次 sudo）
sudo usermod -aG docker $USER
newgrp docker
```

> 若网络访问 docker.com 不畅，可切阿里云镜像源
> `https://mirrors.aliyun.com/docker-ce/linux/ubuntu`。

### 1.2 登录阿里云 ACR

```bash
docker login registry.cn-hangzhou.aliyuncs.com
# Username: aliyun@xxxxxxxxxx（ACR 独立访问用户名）
# Password: ACR 独立密码（NOT 阿里云主账号密码）
```

凭证写入 `~/.docker/config.json`，后续 `docker compose pull` 自动复用。

### 1.3 准备部署目录

```bash
sudo mkdir -p /opt/hemi
sudo chown $USER:$USER /opt/hemi
cd /opt/hemi

# 拉仓库中 deploy/ 下的部署文件
git clone --depth=1 --sparse <GITHUB_REPO_URL> repo
cd repo
git sparse-checkout set deploy
cd /opt/hemi
cp -r repo/deploy/* .
rm -rf repo

ls -la
# 应当能看到：docker-compose.yml  .env.example  Caddyfile
```

> 若不想 git clone 整库，也可以直接 `scp` 三个文件上来。

### 1.4 配置 `.env`

```bash
cp .env.example .env
vi .env
```

**必填**：

| 变量 | 说明 | 生成命令 |
|------|------|---------|
| `IMAGE_TAG` | 完整镜像 URL | `registry.cn-hangzhou.aliyuncs.com/<NS>/hemi-server:latest` |
| `DOMAIN` | 裸域 | `example.com` |
| `PHX_HOST` | Phoenix URL host | `auth.example.com` |
| `SECRET_KEY_BASE` | 64+ 字符 | `openssl rand -base64 64 \| tr -d '\n'` |
| `RELEASE_COOKIE` | 节点 cookie | `openssl rand -base64 48 \| tr -d '\n'` |
| `MMO_DB_PASSWORD` | DB 密码 | `openssl rand -base64 24 \| tr -d '\n'` |

**锁定权限**：

```bash
chmod 600 .env
```

### 1.5 启动

```bash
docker compose pull      # 从 ACR 拉镜像（首次耗时）
docker compose up -d
docker compose ps        # 应看到 postgres + app + caddy 都 running
docker compose logs -f app
```

### 1.6 跑数据库 migration

首次启动后，pg 已经 ready，但库里是空的。在 app 容器内执行：

```bash
docker compose exec app /app/bin/hemi_server eval \
    'DataService.Release.migrate()'
```

> **尚未落地**：`DataService.Release.migrate/0` 需要在 `data_service` 里新增
> release-safe migration helper（不能依赖 Mix）。参考 Phoenix 1.8 官方模板
> 的 `lib/my_app/release.ex`。

临时 workaround：手动 psql 连接后执行仓库里的 migration SQL，或把整个
`apps/data_service` 的源码 mount 进容器跑 `mix ecto.migrate`（只在首次初始化）。

### 1.7 验证

```bash
# HTTPS 健康（Caddy 会自动签发 Let's Encrypt 证书，首次 ~30 秒）
curl -I https://auth.$DOMAIN

# TCP 连通
nc -v gate.$DOMAIN $GATE_TCP_PORT
# 或从客户端尝试连接
```

## 2. 日常升级（最常用）

每次 CI 把新镜像推到 ACR 后：

```bash
cd /opt/hemi
docker compose pull app        # 只拉 app，pg/caddy 不动
docker compose up -d app       # 短暂停机重启
docker compose logs -f app
```

**停机时长**：约 5-15 秒（旧容器 stop + 新容器 start + BEAM 初始化）。

如果要同时更新 compose 配置（docker-compose.yml / Caddyfile / .env）：

```bash
# 先拉最新部署文件（按 1.3 的步骤重做，或手动编辑）
docker compose up -d           # 会 recreate 变化的服务
```

## 3. 回滚

```bash
# 查看可用的历史 tag
docker images | grep hemi-server

# 将 .env 的 IMAGE_TAG 指回上一个版本
vi .env   # IMAGE_TAG=...:sha-abcd1234

docker compose up -d app
```

也可以指定 semver：`IMAGE_TAG=...:v0.1.3`。

## 4. 日志与监控

### 4.1 日志

```bash
# 实时
docker compose logs -f app

# 过去 1 小时
docker compose logs --since 1h app

# Caddy 访问日志
docker compose exec caddy tail -f /data/logs/auth.log
```

### 4.2 容器状态

```bash
docker compose ps
docker stats hemi-app-1    # CPU/内存实时
```

### 4.3 进入容器

```bash
docker compose exec app /app/bin/hemi_server remote     # IEx remote shell
docker compose exec postgres psql -U $MMO_DB_USER $MMO_DB_NAME
```

## 5. 常见问题

| 症状 | 原因 | 处置 |
|------|------|------|
| `docker compose pull` 401 | ACR 登录过期 | `docker login registry.cn-...` 重新登录 |
| Caddy 证书签发失败 | 80 端口未放通 / DNS 未解析 | 检查安全组 + DNS |
| app 启动崩 `SECRET_KEY_BASE missing` | `.env` 未正确加载 | 确认 `env_file: .env` + 文件存在 |
| Phoenix 404 | Caddy 把请求转到了错误后端 | `curl -v http://localhost:4000` 直通测试 |
| TCP 29000 连不上 | 安全组 / compose `ports:` 未声明 | `docker compose ps` 看端口映射 |
| libcluster 报错 UDP bind | `DISABLE_CLUSTER=true` 没生效 | 看 runtime.exs 分支是否走到 |

## 6. 备份

### 6.1 PostgreSQL

```bash
# 手动快照
docker compose exec postgres pg_dump -U $MMO_DB_USER $MMO_DB_NAME \
    | gzip > /opt/hemi/backup/pg-$(date +%Y%m%d-%H%M).sql.gz

# crontab 每日凌晨
0 4 * * * docker compose -f /opt/hemi/docker-compose.yml exec -T postgres \
    pg_dump -U hemi hemi_prod | gzip > /opt/hemi/backup/pg-$(date +\%Y\%m\%d).sql.gz
```

### 6.2 Caddy 证书

`caddy_data` volume 持久化 Let's Encrypt 证书。服务器换机时：

```bash
docker run --rm -v hemi_caddy_data:/src -v $PWD:/dst alpine \
    tar czf /dst/caddy-data.tgz -C /src .
```

## 7. 安全 checklist（首次上线前）

- [ ] `.env` 权限 600，只 root/部署用户可读
- [ ] 服务器 SSH 关掉密码，只留 key 登录
- [ ] 安全组最小开放（80/443/${GATE_TCP_PORT}），禁止 5432 公网
- [ ] `visualize_server` 加 IP 白名单或 basic auth（Caddyfile 中注释了位置）
- [ ] Phoenix 的 admin / dashboard 路由确认未在生产暴露
- [ ] `docker compose logs` 中确认没有 leak 环境变量
- [ ] 第一次上线后 `curl -I https://auth.$DOMAIN` 返回 200 / 404，不是 5xx
