# 2026-04-17 服务器部署手册（ex_mmo_cluster — 单容器 MVP）

对应架构文档：`docs/2026-04-17-上线-Docker-CI-部署方案.md`

## 0. 前提

- 一台 Ubuntu 22.04 或 24.04 的阿里云/国内服务器（≥ 4C 8G）
- 宿主机已装 **nginx**（用于 HTTPS 反向代理；本手册不重复安装步骤）
- 已解析到该服务器的域名 `$DOMAIN`，一条 A 记录即可（如 `mmo.hemifuture.cn`）
  - HTTPS（auth_server）走 `$DOMAIN:443`
  - TCP 游戏网关（gate_server）走 `$DOMAIN:${GATE_TCP_PORT}`（也可直接用 IP）
  - visualize_server（4001）**不暴露公网**，运维用 SSH 隧道访问
- 服务器安全组放通：`80/tcp`、`443/tcp`、`${GATE_TCP_PORT}/tcp`（默认 29000）
- 阿里云 ACR 账号已可登录（CI 推送用的相同凭证即可）

## 1. 首次初始化

### 1.1 装 Docker 与 compose 插件

```bash
# Ubuntu 官方源
sudo apt-get update
sudo apt-get install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
    -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] \
    https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo $VERSION_CODENAME) stable" | \
    sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io \
    docker-buildx-plugin docker-compose-plugin

# 当前用户加入 docker 组（避免每次 sudo）
sudo usermod -aG docker $USER
newgrp docker
```

> 若网络访问 docker.com 不畅，可切阿里云镜像源
> `https://mirrors.aliyun.com/docker-ce/linux/ubuntu`。

### 1.2 登录阿里云 ACR

```bash
docker login registry.cn-hangzhou.aliyuncs.com
# Username: aliyun@xxxxxxxxxx（ACR 独立访问用户名）
# Password: ACR 独立密码（NOT 阿里云主账号密码）
```

凭证写入 `~/.docker/config.json`，后续 `docker compose pull` 自动复用。

### 1.3 准备部署目录

```bash
sudo mkdir -p /opt/ex_mmo_cluster
sudo chown $USER:$USER /opt/ex_mmo_cluster
cd /opt/ex_mmo_cluster

# 拉仓库中 deploy/ 下的部署文件
git clone --depth=1 --sparse <GITHUB_REPO_URL> repo
cd repo
git sparse-checkout set deploy
cd /opt/ex_mmo_cluster
cp -r repo/deploy/* .
rm -rf repo

ls -la
# 应当能看到：docker-compose.yml  .env.example  nginx.conf.example
```

> 若不想 git clone 整库，也可以直接 `scp` 三个文件上来。

### 1.4 配置 `.env`

```bash
cp .env.example .env
vi .env
```

**必填**：

| 变量 | 说明 | 生成命令 |
|------|------|---------|
| `IMAGE_TAG` | 完整镜像 URL | `registry.cn-hangzhou.aliyuncs.com/<NS>/ex_mmo_cluster:latest` |
| `DOMAIN` | 裸域 | `example.com` |
| `PHX_HOST` | Phoenix URL host（与 `DOMAIN` 一致） | `mmo.example.com` |
| `SECRET_KEY_BASE` | 64+ 字符 | `openssl rand -base64 64 \| tr -d '\n'` |
| `RELEASE_COOKIE` | 节点 cookie | `openssl rand -base64 48 \| tr -d '\n'` |
| `MMO_DB_PASSWORD` | DB 密码 | `openssl rand -base64 24 \| tr -d '\n'` |

**锁定权限**：

```bash
chmod 600 .env
```

### 1.5 配置宿主 nginx

证书已经放在 `/etc/nginx/ssl/$DOMAIN/cert.pem` + `key.pem`（自管，非 Let's Encrypt）。
模板默认就指这两个路径，无需调整。

```bash
# 1) 确认证书与权限
sudo ls -l /etc/nginx/ssl/$DOMAIN/
# 应看到 cert.pem 和 key.pem；key.pem 建议 600，owner 为 root

# 2) 把模板拷到 nginx 配置目录
sudo cp /opt/ex_mmo_cluster/nginx.conf.example /etc/nginx/sites-available/ex_mmo_cluster.conf
sudo sed -i "s/\${DOMAIN}/$DOMAIN/g" /etc/nginx/sites-available/ex_mmo_cluster.conf
sudo ln -sf /etc/nginx/sites-available/ex_mmo_cluster.conf /etc/nginx/sites-enabled/

# 3) 验证 + reload
sudo nginx -t
sudo systemctl reload nginx
```

> 续期由证书来源（手动/脚本/外部签发服务）自行管理；nginx 重新读取证书需要
> `sudo systemctl reload nginx`。若后续改回 Let's Encrypt，参考 `nginx.conf.example`
> 顶部注释里给出的 `/etc/letsencrypt/live/...` 路径替换。

### 1.6 启动容器

```bash
docker compose pull      # 从 ACR 拉镜像（首次耗时）
docker compose up -d
docker compose ps        # 应看到 postgres + app 都 running
docker compose logs -f app
```

### 1.7 跑数据库 migration

首次启动后，pg 已经 ready，但库里是空的。在 app 容器内执行：

```bash
docker compose exec app /app/bin/ex_mmo_cluster eval \
    'DataService.Release.migrate()'
```

> **尚未落地**：`DataService.Release.migrate/0` 需要在 `data_service` 里新增
> release-safe migration helper（不能依赖 Mix）。参考 Phoenix 1.8 官方模板
> 的 `lib/my_app/release.ex`。

临时 workaround：手动 psql 连接后执行仓库里的 migration SQL，或把整个
`apps/data_service` 的源码 mount 进容器跑 `mix ecto.migrate`（只在首次初始化）。

### 1.8 验证

```bash
# HTTPS 健康（证书在 1.5 步骤已就位）
curl -I https://$DOMAIN

# TCP 连通（游戏网关）
nc -v $DOMAIN $GATE_TCP_PORT
# 或从客户端尝试连接

# Visualize 控制台（通过 SSH 隧道访问）
ssh -L 4001:127.0.0.1:4001 user@$DOMAIN
# 浏览器打开 http://localhost:4001
```

## 2. 日常升级（最常用）

每次 CI 把新镜像推到 ACR 后：

```bash
cd /opt/ex_mmo_cluster
docker compose pull app        # 只拉 app，pg 不动
docker compose up -d app       # 短暂停机重启
docker compose logs -f app
```

**停机时长**：约 5-15 秒（旧容器 stop + 新容器 start + BEAM 初始化）。

如果要同时更新 compose 配置（docker-compose.yml / .env）或宿主 nginx 配置：

```bash
# 先拉最新部署文件（按 1.3 的步骤重做，或手动编辑）
docker compose up -d           # 会 recreate 变化的服务
```

## 5. 常见问题

| 症状 | 原因 | 处置 |
|------|------|------|
| `docker compose pull` 401 | ACR 登录过期 | `docker login registry.cn-...` 重新登录 |
| nginx 启动报 `cannot load certificate` | `/etc/nginx/ssl/$DOMAIN/{cert,key}.pem` 路径 / 权限不对 | `ls -l /etc/nginx/ssl/$DOMAIN/`，确认两个文件存在且 nginx 可读 |
| 浏览器 SSL 报"证书过期" | 自管证书未及时续 | 重新签发 → 覆盖 `cert.pem` / `key.pem` → `sudo systemctl reload nginx` |
| app 启动崩 `SECRET_KEY_BASE missing` | `.env` 未正确加载 | 确认 `env_file: .env` + 文件存在 |
| Phoenix 404 / 502 | nginx 反代到了错的端口 | `curl -v http://127.0.0.1:4000` 直通测试，再核对 `nginx.conf` 的 `proxy_pass` |
| TCP 29000 连不上 | 安全组 / compose `ports:` 未声明 | `docker compose ps` 看端口映射 |
| libcluster 报错 UDP bind | `DISABLE_CLUSTER=true` 没生效 | 看 runtime.exs 分支是否走到 |

## 6. 备份

### 6.1 PostgreSQL

```bash
# 手动快照
docker compose exec postgres pg_dump -U $MMO_DB_USER $MMO_DB_NAME \
    | gzip > /opt/ex_mmo_cluster/backup/pg-$(date +%Y%m%d-%H%M).sql.gz

# crontab 每日凌晨
0 4 * * * docker compose -f /opt/ex_mmo_cluster/docker-compose.yml exec -T postgres \
    pg_dump -U ex_mmo_cluster ex_mmo_cluster_prod | gzip > /opt/ex_mmo_cluster/backup/pg-$(date +\%Y\%m\%d).sql.gz
```

### 6.2 TLS 证书

自管证书放在 `/etc/nginx/ssl/$DOMAIN/{cert,key}.pem`。服务器换机或灾备：

```bash
sudo tar czf /opt/ex_mmo_cluster/backup/nginx-ssl-$(date +%Y%m%d).tgz \
    -C /etc/nginx ssl
```

恢复：解包到新机器的 `/etc/nginx/ssl/`，确认 `key.pem` 权限为 600 / owner root，
再 `sudo systemctl reload nginx`。

> 证书续期不在本流程里——由签发/托管侧负责。替换文件后必须 `reload nginx`，
> 否则 nginx 仍持有旧证书。

## 7. 安全 checklist（首次上线前）

- [ ] `.env` 权限 600，只 root/部署用户可读
- [ ] 服务器 SSH 关掉密码，只留 key 登录
- [ ] 安全组最小开放（80/443/${GATE_TCP_PORT}），禁止 5432 公网
- [ ] `visualize_server` 未反代到公网 nginx（方案 A：只绑 `127.0.0.1:4001`，运维走 SSH 隧道访问）
- [ ] Phoenix 的 admin / dashboard 路由确认未在生产暴露
- [ ] `docker compose logs` 中确认没有 leak 环境变量
- [ ] 第一次上线后 `curl -I https://$DOMAIN` 返回 200 / 404，不是 5xx
