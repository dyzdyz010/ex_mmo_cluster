# Ex MMO Cluster 项目概览

**更新日期：** 2026-04-10

## 项目定位

`ex_mmo_cluster` 是一个用于探索 MMORPG 风格分布式服务架构的 Elixir Umbrella 项目。当前仓库已经完成一部分基础迁移工作，但整体仍处于**迁移中、能力不均衡**的状态。

## 当前现实情况

目前仓库的实际情况可以概括为：

- 本地 `mix compile` 与 `mix test` 已可通过
- `gate_server`、`scene_server`、`beacon_server` 已有比较实质的运行时代码
- `auth_server` 与 `gate_server` 的 token 验证链路已经打通，但 auth 业务逻辑仍未完成
- 持久化仍是 **PostgreSQL + 旧 Mnesia 组件并存** 的混合阶段
- `world_server`、`agent_server`、`visualize_server` 仍然明显轻量，更多是骨架或实验性入口

## Umbrella 应用结构

```text
client
  └─ gate_server
       ├─ auth_server
       ├─ scene_server
       └─ （游戏行为流仍未完全收口）

scene_server
  └─ world_server

auth_server / world_server
  └─ data_service

agent_server
  └─ agent_manager

service discovery
  └─ beacon_server

legacy / migration-era data topology
  ├─ data_contact
  ├─ data_store
  └─ data_init

ui / inspection shell
  └─ visualize_server
```

## 当前应用分层

### 核心运行时

- `gate_server` —— TCP 入口、自定义二进制协议、scene/auth 分发
- `scene_server` —— 场景运行时、AOI、物理与角色运动循环
- `beacon_server` —— 基于 `libcluster + Horde` 的服务发现与集群注册

### 已经有真实能力，但还在收敛

- `auth_server` —— Web 登录入口 + 集群可见 token 验证面
- `data_service` —— Ecto/PostgreSQL 已接入，但仍带有旧拓扑启动依赖
- `visualize_server` —— 面向开发者的 LiveView 观察壳层

### 骨架/占位成分仍然较重

- `world_server` —— 世界层协调职责仍然很薄
- `agent_server` —— agent / 玩家域逻辑仍偏原型态
- `agent_manager` —— 协调入口存在，但调度能力尚未真正落地

### 迁移期遗留支撑

- `data_contact` —— 旧数据拓扑中的 registry / selector
- `data_store` —— 旧 Mnesia store 节点
- `data_init` —— 旧 Mnesia schema/bootstrap 工具

## 当前迁移状态

根据 [`2026-04-07-增量迁移计划.md`](2026-04-07-增量迁移计划.md)，仓库已经完成：

1. TCP 分帧 + 自定义二进制协议迁移
2. PostgreSQL / Ecto 基础接入
3. `beacon_server` 高可用迁移（`libcluster + Horde`）

仍待完成：

1. Scene 空间分区
2. 玩家状态持久化与崩溃恢复
3. 完整移除 Mnesia 时代表面
4. world / agent / visualization 的产品化深化

## 本地开发

### 工具链

当前仓库预期使用：

- Erlang/OTP `28.3.1`
- Elixir `1.18.4-otp-28`

版本固定见 `.tool-versions`。

### 常用命令

```bash
mix deps.get
mix compile
mix test
```

## 当前注意事项

- 仓库还没有完成向 PostgreSQL-only 持久化的最终切换
- 一部分 app 的实现已经是真实运行时能力，一部分仍然偏骨架
- 测试通过说明当前仓库状态具备本地可复现性，不代表所有 MMO 子系统都已功能完备
