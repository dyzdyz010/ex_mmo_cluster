# 2026-04-13 移动同步 vNext 阶段一完成记录

## 本轮完成

### 1. 本地输入流与权威 ack 已成为运行时主路径
- 客户端高频移动发送 `MovementInput`
  - `seq`
  - `client_tick`
  - `dt_ms`
  - `input_dir`
  - `speed_scale`
  - `movement_flags`
- 服务端返回 `MovementAck`
  - `ack_seq`
  - `auth_tick`
  - `position`
  - `velocity`
  - `acceleration`
  - `movement_mode`
  - `correction_flags`
- 客户端运行时已接入：
  - `InputHistory`
  - `PredictedHistory`
  - `Predictor`
  - `Reconcile`

### 2. 远端移动广播已升级为 vNext snapshot
- `PlayerMove` 不再只广播位置与 sequence。
- 当前广播内容为：
  - `cid`
  - `server_tick`
  - `position`
  - `velocity`
  - `acceleration`
  - `movement_mode`
- Scene AOI 广播现在通过 `SceneServer.Movement.RemoteSnapshot` 传递。
- Gate codec / TCP / UDP 下行都已切到新 snapshot 语义。

### 3. 客户端远端角色路径已接入 snapshot-aware 表现层
- 新增 `world/remote_player.rs`
- 客户端保存远端角色 snapshot 状态而不是裸位置。
- 渲染层会基于：
  - `position`
  - `velocity`
  - `acceleration`
  做短时外推采样，再交给可视层 smoothing。
- 这让远端角色移动表现不再只依赖离散位置点。

### 4. demo bot 已切到 vNext movement input
- Demo bot 不再发送旧的“位置提议 movement”。
- 当前改为：
  - 基于 waypoint 计算方向向量
  - 发送 `MovementInput`
  - 接收 `MovementAck`
  - 维护自己的权威位置
- 这让 demo / E2E 使用的 bot 与真实客户端走同一条核心移动链路。

### 5. E2E harness 已修正为等待真实 runtime 就绪
- `scripts/e2e-stdio.ps1` 现在：
  - 使用独立 output/observe 目录
  - 避免复用旧配置文件
  - 等待 demo bot AOI 流量出现后再继续
  - 校验 server/client stdio 输出中的真实连接、fast-lane、remote player
- 这样 E2E 不再因为 demo.run 启动/seed 时序而误判。

## 已删除/收敛

- 删除了仅为旧移动路径保留的 movement tick 定时器逻辑。
