## 本轮完成

### 1. 服务端新增 combat domain

新增：

- `apps/scene_server/lib/scene_server/combat/profile.ex`
- `apps/scene_server/lib/scene_server/combat/state.ex`
- `apps/scene_server/lib/scene_server/combat/skill.ex`
- `apps/scene_server/lib/scene_server/combat/targeting.ex`

这些模块分别负责：

- `Combat.Profile`
  - HP 上限
  - respawn 时间
- `Combat.State`
  - hp / max_hp / alive / deaths
  - damage / kill / respawn 逻辑
- `Combat.Skill`
  - `skill_id=1` 的权威定义
  - damage / radius / cooldown
- `Combat.Targeting`
  - 基于 AOI / 空间查询获取候选目标
  - 距离过滤

### 2. PlayerCharacter 结构继续完善

`SceneServer.PlayerCharacter` 现在不再只有 movement state，而是同时维护：

- movement state
- combat profile
- combat state
- spawn location
- respawn timer
- skill cooldown state

并新增：

- `:get_state_summary`
- `{:apply_skill_hit, ...}`
- `:respawn`

这让玩家 GenServer 真正开始承担：

- 移动
- 生存状态
- 技能命中处理
- respawn 生命周期

### 3. 最小技能闭环

当前 `skill_id=1`：

- 有权威 radius
- 有 damage
- 有 cooldown

触发时流程：

1. 客户端发 `SkillCast`
2. 服务端 `PlayerCharacter` 校验：
   - 活着
   - cooldown
   - 技能存在
3. 通过 `Combat.Targeting` 找到目标
4. 目标 `PlayerCharacter` 执行 `apply_skill_hit`
5. 更新 HP
6. 广播：
   - `SkillEvent`
   - `CombatHit`
   - `PlayerState`

### 4. 新增 player_state / combat_hit 广播

新增线协议消息：

- `0x8C` `PlayerState`
- `0x8D` `CombatHit`

服务端：

- `GateServer.Codec`
- `GateServer.TcpConnection`

客户端：

- `clients/bevy_client/src/protocol.rs`
- `clients/bevy_client/src/net.rs`

都已接入。

### 5. AOI / 广播链路

`AoiItem` 现在支持：

- `player_state`
- `health_update`
- `combat_hit`
- `combat_resolved`

并修复了 combat 广播的递归风暴问题：

- 发起方用 `combat_resolved`
- 订阅方只收到 `combat_hit`

避免了循环广播。

### 6. 客户端表现与 HUD

客户端现在支持：

- 本地 HP / alive 状态
- `CombatHit` 日志
- `PlayerState` 更新
- snapshot/stdio 中也会输出：
  - `local_hp`
  - `local_max_hp`
  - `local_alive`
