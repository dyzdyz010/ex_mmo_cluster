# DataService 应用说明

**更新日期：** 2026-04-10

## 当前定位

`data_service` 是数据层唯一的持久化主路径，基于 Ecto + PostgreSQL。旧 Mnesia 链路（`data_init` / `data_store` / `data_contact`、`DataService.Dispatcher` + poolboy 串行层）已于 2026-06-04 全部物理删除。

## 当前已实现

- `DataService.Repo`（Ecto + PostgreSQL）
- `DataService.Worker`：无状态账户/角色访问函数（`auth_server` 同 BEAM 直调）
- 基于 Ecto 的账户注册、查询、dev upsert 路径
- 体素相关 store（map ledger / chunk snapshot / scene object 等）

## 开发提示

- 数据访问统一走 `DataService.Repo` / `DataService.Worker`；不要再引入 Mnesia/Memento
- 跨节点账户访问由 `auth_server` 边界负责，data_service 自身只暴露无状态 Ecto 调用
